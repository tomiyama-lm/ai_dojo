各回の詳細内容

01

Cursor で今すぐ開発効率3倍アップ

目標：明日から使えるCursor活用法をマスターし、開発スピードの向上を体感する

実践ハンズオン（45分）

  • 5分で体感する開発効率化

    簡単なTodoリストコンポーネントを従来方法とCursorで比較作成

  • 業務でよくある場面での活用

    バグ修正、機能追加、リファクタリング、API連携の実践

  • Cursor基本操作の実習

    Ctrl+K（Inline edit）、Chat、@ファイル指定の活用

ミニ講義（15分）

  • なぜこんなに便利なのか
  • Cursorが普通のエディタと違う理由
  • コード理解の仕組み（簡単に）
  • 他のAIツールとの使い分け
今日の成果と明日への活用

参加者全員が「明日やってみたいこと」を1つ決め、実際の業務コードでの活用チャレンジを設定

02

「伝え方」で結果が10倍変わる プロンプト実践術

目標：具体的な業務シーンでのプロンプト技術を身につけ、AIから期待する答えを引き出せるようになる

実況中継ハンズオン（40分）

  • Before/After で体感する差

    悪いプロンプト vs 良いプロンプトの実際の違いを体験

  • 業務シーン別プロンプト実践

    • コードレビュー依頼

    • エラー解決

    • 設計相談

    • テストコード作成

  • リアルタイム改善実習

    参加者の実際の課題を持ち寄り、その場でプロンプトを改善

パターン整理（20分）

  • 効果的なプロンプトの共通パターン

    • 役割設定、制約条件、出力形式の指定方法

    • 具体例の提示（Few-shot）の使い方

    • 段階的な質問の組み立て方

自分用テンプレート作成

各自の業務でよく使うプロンプトテンプレートを3個作成し、チーム内で共有

03

複数AIツール連携で業務をフルオートメーション化

目標：複数のAIサービスを組み合わせて、複雑な業務タスクを効率化する

連携実践ワークショップ（50分）

画像からコードを生成する流れ

手書きモックアップ → Geminiで解析 → Cursorで実装

ドキュメント作業の自動化

仕様書PDF → 要約作成 → 実装タスクの洗い出し

チーム開発での活用

GitHub Issue → 技術調査 → 実装方針提案

API活用の基礎

ChatGPT API、Claude APIの簡単な使い方

ツールチェーン設計（10分）

自分の業務の中で「この流れを自動化できそう」なワークフローを1つ設計し、次回までに実際に試してもらう

04

なぜAIがこんなに賢いのか？仕組み理解で使い方が変わる

目標：実践で感じた「なぜ？」を理論で理解し、より効果的な活用法を身につける

体験ベース理論学習（30分）

これまでの実践を振り返りながら
  • • なぜCursorはコードの文脈を理解できるの？
  • • なぜプロンプトによって回答の質が変わるの？
  • • なぜ同じ質問でも回答が毎回違うの？
LLMの基本の仕組み
  • • トークン化：AIが文章をどう理解しているか
  • • 文脈理解：なぜ長い会話ができるのか
  • • 確率的生成：なぜ時々間違えるのか
各AIサービスの特徴と使い分け
  • • GPT-4：汎用性の高さ
  • • Claude：長文処理・分析が得意
  • • Gemini：多言語・リアルタイム情報
  • • GitHub Copilot：コード特化

理論を活かした実践（30分）

理解に基づく活用方法の改善
  • • コンテキスト長を意識したプロンプト設計
  • • 温度パラメータの調整（創造性 vs 正確性）
  • • トークン制限を考慮した効率的な質問方法
AIの限界と対策
  • • ハルシネーション（幻覚）への対処法
  • • 古い情報への対策
  • • セキュリティ・プライバシーの注意点
応用課題

理論を踏まえて、第2回で作ったプロンプトテンプレートを改良
